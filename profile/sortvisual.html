Selection Sort
Find smallest value from left(start after sorted elements) to right and update then repeat until it reach 2nd last element
Insertion Sort
Insert value from right to left until it is smaller than the left of it
Bubble Sort
From the left find greatest value and move it to the end right before the already sorted values then repeat
Merge Sort
Recusive spliting into 2 half pieces and then back merging the 2 pieces
Quick Sort
Pick a value from the most right then move all values to right of it if it is smallers and to the left if it greater then repeat recusively its left and right sides of it

Results:
 Quick Sort is the fastest and comparable to merges sort since
they both are doing N time comparation for each of half
piece data size that result in N*log(n) run time in total while
other sorts have to check all the elements and do the comparation
for a whole size data size		 In general Merge Sort runs fastest it spliting data into many
half pieces and at the smallest size it is automaticall sorted
and the merging only take N time because the 2 half pieces are
already sorted therefore total runtime is Nlog(N) time		 Insertion Sort is the best since it only need to check 1 value
before it and skip ALL insertion steps
Conclusion: Merge Sort is the best regardless to whether data is unique, unsorted, sorted, or various data type. It always do the similar fast run time which is N(for merging) * logN (spliting)
